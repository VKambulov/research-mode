## Summary

Collected primary and official sources for the first search pass: robots.txt/REP, HTTP safe methods and redirects, URL parsing/security, SSRF defenses, Markdown/raw HTML behavior, MarkItDown conversion caveats, and prompt-injection handling for untrusted web content.

## Sources

- [RFC 9309: Robots Exclusion Protocol](https://datatracker.ietf.org/doc/html/rfc9309) — Primary standard for robots.txt behavior, matching rules, redirects, caching, and the warning that robots.txt is not access authorization.
- [RFC 9110: HTTP Semantics](https://datatracker.ietf.org/doc/html/rfc9110) — Primary HTTP semantics source for safe methods, redirects, Location handling, and User-Agent guidance.
- [Python urllib.parse documentation](https://docs.python.org/3/library/urllib.parse.html) — Official URL parsing documentation with security warnings about validation and urljoin with attacker-controlled input.
- [OWASP Server-Side Request Forgery Prevention Cheat Sheet](https://cheatsheetseries.owasp.org/cheatsheets/Server_Side_Request_Forgery_Prevention_Cheat_Sheet.html) — Official OWASP defensive guidance for URL-controlled fetches, allowlists, parser abuse, and network-layer containment.
- [OWASP LLM Prompt Injection Prevention Cheat Sheet](https://cheatsheetseries.owasp.org/cheatsheets/LLM_Prompt_Injection_Prevention_Cheat_Sheet.html) — Official OWASP guidance for treating external content as untrusted, action screening, least privilege, and quarantine/dual-model patterns.
- [CommonMark Spec 0.31.2](https://spec.commonmark.org/0.31.2/) — Primary Markdown syntax source, useful for noting that Markdown may include HTML blocks and raw HTML inlines depending on rendering.
- [Microsoft MarkItDown repository](https://github.com/microsoft/markitdown) — Official converter documentation that frames Markdown conversion for LLM/text analysis and warns that conversion performs I/O with current-process privileges.

## Findings

- **fact**: robots.txt should be treated as a crawler politeness boundary, not as authorization or proof that captured content is safe to republish; RFC 9309 explicitly notes that robots.txt is not a substitute for real access controls. (refs: https://datatracker.ietf.org/doc/html/rfc9309)
- **fact**: A safe capture workflow should prefer HTTP safe retrieval semantics such as GET/HEAD and avoid triggering unsafe actions; RFC 9110 defines safe methods as essentially read-only from the client's perspective. (refs: https://datatracker.ietf.org/doc/html/rfc9110)
- **risk**: User-provided URLs are an SSRF and redirect risk. The example should require scheme/host validation, internal-address blocking, redirect revalidation, bounded redirects, timeouts, size limits, and network egress controls rather than trusting parser output alone. (refs: https://docs.python.org/3/library/urllib.parse.html, https://cheatsheetseries.owasp.org/cheatsheets/Server_Side_Request_Forgery_Prevention_Cheat_Sheet.html, https://datatracker.ietf.org/doc/html/rfc9110)
- **risk**: Fetched web pages and converted Markdown are untrusted model input. The agent workflow should quarantine raw captures, delimit or structure external content, prevent embedded page instructions from driving tools, and screen any proposed action against the original user intent. (refs: https://cheatsheetseries.owasp.org/cheatsheets/LLM_Prompt_Injection_Prevention_Cheat_Sheet.html)
- **risk**: Markdown is not automatically safe output: CommonMark includes raw HTML constructs, and converters may preserve links or HTML-like content. A public example should separate captured Markdown from rendered HTML, document sanitization assumptions, and retain source metadata. (refs: https://spec.commonmark.org/0.31.2/, https://github.com/microsoft/markitdown)
- **plan**: The requested example package can be organized around five reusable artifacts: README for purpose and safety boundaries, prompt.md for the worker instruction contract, research-plan.md for stages and gates, sources.md for official references, and final-report.md for a concise evaluated workflow and artifact checklist. (refs: https://datatracker.ietf.org/doc/html/rfc9309, https://datatracker.ietf.org/doc/html/rfc9110, https://cheatsheetseries.owasp.org/cheatsheets/Server_Side_Request_Forgery_Prevention_Cheat_Sheet.html, https://cheatsheetseries.owasp.org/cheatsheets/LLM_Prompt_Injection_Prevention_Cheat_Sheet.html)
