## Summary

Seeded the research with five official-source candidates for the RAG evaluation tooling matrix: Ragas, DeepEval, Phoenix, LangSmith, and Promptfoo. Captured their practical positioning around metric-driven eval loops, local/CI testing, observability plus RAG eval, managed dataset/experiment workflows, and lightweight CLI-based RAG tests.

## Sources

- [Ragas documentation](https://docs.ragas.io/en/stable/) — Official docs describe Ragas as a library for systematic evaluation loops, LLM-app metrics, test set generation, and integrations with frameworks such as LangChain and LlamaIndex.
- [DeepEval 5-min Quickstart](https://deepeval.com/docs/getting-started) — Official quickstart shows local evaluation through test cases, metrics, and the deepeval test run workflow, with optional Confident AI reporting.
- [Phoenix: Evaluate RAG](https://arize.com/docs/phoenix/cookbook/evaluation/evaluate-rag) — Official Phoenix cookbook demonstrates building a RAG pipeline, tracing it, and evaluating retrieval and response quality with Phoenix Evals.
- [LangSmith: Evaluate a RAG application](https://docs.langchain.com/langsmith/evaluate-rag-tutorial) — Official LangSmith tutorial covers datasets, running a RAG app over examples, and metrics such as correctness, relevance, groundedness, and retrieval relevance.
- [Promptfoo: Evaluating RAG pipelines](https://www.promptfoo.dev/docs/guides/evaluate-rag/) — Official guide frames RAG evaluation as separate retrieval and generation checks, with output-based, context-based, and custom metrics usable from CLI/config workflows.

## Findings

- **fact**: Ragas should be considered a RAG-focused evaluation framework candidate because its official docs emphasize systematic LLM-app eval loops, custom and built-in metrics, test set generation, and common framework integrations. (refs: https://docs.ragas.io/en/stable/)
- **fact**: DeepEval is a strong CI/regression-testing candidate: its official quickstart is structured around local test cases, evaluation metrics, and a deepeval test run command, with optional hosted reporting through Confident AI. (refs: https://deepeval.com/docs/getting-started)
- **fact**: Phoenix belongs in the observability-plus-evaluation category because the RAG cookbook combines tracing/instrumentation with retrieval and response evaluation. (refs: https://arize.com/docs/phoenix/cookbook/evaluation/evaluate-rag)
- **fact**: LangSmith represents a managed experiment/dataset workflow for RAG evaluation: the official tutorial organizes evaluation around datasets, running the app on those datasets, and measuring answer relevance, accuracy, and retrieval quality. (refs: https://docs.langchain.com/langsmith/evaluate-rag-tutorial)
- **fact**: Promptfoo is a lightweight CLI/config-oriented candidate for production teams that want to test retrieval and generation separately and wire RAG checks into repeatable automation. (refs: https://www.promptfoo.dev/docs/guides/evaluate-rag/)
- **plan**: A useful XLSX scoring model should likely separate tool type, license/hosting posture, RAG-specific metric coverage, CI automation, trace/observability support, dataset/testset support, human feedback/annotation, integration footprint, operational maturity, and evidence quality. (refs: https://docs.ragas.io/en/stable/, https://deepeval.com/docs/getting-started, https://arize.com/docs/phoenix/cookbook/evaluation/evaluate-rag, https://docs.langchain.com/langsmith/evaluate-rag-tutorial, https://www.promptfoo.dev/docs/guides/evaluate-rag/)
- **gap**: The current source set is a seed shortlist, not yet enough for final comparison. It still needs coverage of Langfuse, Opik, TruLens, MLflow GenAI, Giskard, Evidently, Weave, LlamaIndex, Haystack, and OpenAI Evals before synthesis.
