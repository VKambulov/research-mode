## Findings

- **fact**: Langfuse is a strong primary-row candidate for teams that want observability-first RAG evaluation: its official cookbook shows scores attached to traces/spans, segmentation and analytics over scores, and Ragas evaluation over production traces without requiring ground-truth answers. (refs: https://langfuse.com/guides/cookbook/evaluation_of_rag_with_ragas)
- **fact**: Opik is a strong primary-row candidate for a production engineering matrix because the official repository documents tracing, datasets and experiments, LLM-as-judge metrics for hallucination/moderation/RAG assessment, PyTest CI integration, production dashboards, online evaluation rules, cloud use, and self-hosting. (refs: https://github.com/comet-ml/opik)
- **fact**: TruLens should be scored highly for RAG-specific evaluation concepts: its RAG Triad explicitly separates context relevance, groundedness, and answer relevance, which maps directly to retrieval, grounding, and final-answer quality dimensions in the workbook. (refs: https://www.trulens.org/getting_started/core_concepts/rag_triad/)
- **fact**: MLflow GenAI evaluation is a primary-row candidate for teams already standardizing on MLflow or needing broader AI engineering lifecycle support: official docs list evaluation datasets, human feedback, LLM-as-a-judge scorers, systematic evaluation, tracing, and production monitoring. (refs: https://mlflow.org/docs/latest/genai/eval-monitor/)
- **fact**: Giskard is a primary-row candidate for test-suite-driven RAG validation: official docs show a comprehensive RAG test suite spanning retrieval quality, groundedness, answer quality, out-of-scope questions, and multi-turn conversational RAG. (refs: https://docs.giskard.ai/oss/checks/use-cases/rag-evaluation)
- **synthesis**: A fair public-source scoring model should separate capability from packaging: proposed dimensions are RAG-specific metric coverage, repeatable regression/CI workflow, trace/observability integration, human review and feedback workflow, deployment/governance fit, integration maturity, and evidence maturity. (refs: https://langfuse.com/guides/cookbook/evaluation_of_rag_with_ragas, https://github.com/comet-ml/opik, https://www.trulens.org/getting_started/core_concepts/rag_triad/, https://mlflow.org/docs/latest/genai/eval-monitor/, https://docs.giskard.ai/oss/checks/use-cases/rag-evaluation)
- **gap**: The candidate list is not yet complete for the requested workbook: Evidently, Weave, LlamaIndex, Haystack, and OpenAI Evals still need official-source evidence and classification before analysis/synthesis.
