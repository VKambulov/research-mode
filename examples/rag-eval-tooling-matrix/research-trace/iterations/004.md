## Findings

- **synthesis**: The workbook should separate evaluation capability scoring from deployment/governance scoring. Several candidates have strong OSS/local evaluation capability but enterprise controls only in a paired commercial platform. (refs: https://github.com/confident-ai/deepeval, https://deepeval.com/enterprise, https://github.com/Giskard-AI/giskard-oss, https://www.giskard.ai/)
- **fact**: LangSmith has publicly documented enterprise deployment options covering managed cloud, hybrid deployment with customer VPC data plane, and self-hosted Docker Compose or Kubernetes, plus SSO/JIT/SCIM and role management. (refs: https://docs.langchain.com/langsmith/enterprise)
- **fact**: LangSmith trace retention is publicly documented as Base 14 days and Extended 400 days, with enterprise customization for extended retention per workspace. (refs: https://docs.langchain.com/langsmith/administration-overview)
- **fact**: Langfuse has public evidence for RBAC roles/scopes and data retention, including self-hosted retention requirements for S3-compatible blob storage deletion. (refs: https://langfuse.com/docs/administration/rbac, https://langfuse.com/docs/administration/data-retention)
- **fact**: Phoenix has public self-hosted governance evidence: authentication-enabled instances support admin/member/viewer roles, and self-hosted Phoenix states trace, evaluation, dataset, and application data stays within the user's infrastructure. (refs: https://arize.com/docs/phoenix/self-hosting/features/authentication, https://arize.com/docs/phoenix/self-hosting/security/privacy)
- **fact**: W&B Weave supports a self-managed deployment path, but official docs show it is an enterprise-grade deployment requiring W&B Platform, a Weave-enabled license, Kubernetes, ClickHouse, and S3-compatible storage. (refs: https://docs.wandb.ai/weave/guides/platform/weave-self-managed)
- **risk**: Evidently's current official open-source-vs-cloud FAQ should be treated carefully: it documents OSS Apache-2.0 libraries and platform editions, but also contains confusing wording around whether Evidently Cloud is currently available as SaaS. (refs: https://docs.evidentlyai.com/faq/oss_vs_cloud)
- **fact**: MLflow is publicly documented as fully open-source and self-hostable, with self-hosted authentication and APIs for managing users and permissions. (refs: https://mlflow.org/docs/latest/self-hosting/, https://mlflow.org/docs/latest/self-hosting/security/basic-http-auth/)
- **risk**: Promptfoo should be scored high for local/private/CI-oriented evals but should carry an execution-risk note: its OSS security model explicitly says it is a local eval runner, not a sandbox for adversarial or untrusted configs. (refs: https://github.com/promptfoo/promptfoo/blob/main/SECURITY.md)
- **plan**: Recommended governance workbook columns: license_or_commercial_posture, deployment_options, self_hosting_evidence, managed_saas_evidence, access_control_evidence, retention_or_data_control_evidence, enterprise_controls_evidence, governance_confidence, and governance_notes. (refs: https://docs.langchain.com/langsmith/enterprise, https://langfuse.com/docs/administration/rbac, https://arize.com/docs/phoenix/self-hosting/security/privacy, https://docs.wandb.ai/weave/guides/platform/weave-self-managed)
