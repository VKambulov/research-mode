## Findings

- **synthesis**: The workbook should represent OSS libraries with optional commercial platforms as one row per tool family by default, using split deployment/governance notes rather than separate rows. This fits DeepEval/Confident AI, Giskard OSS/Giskard Hub, and Evidently OSS/Platform unless the hosted product has materially different evaluator coverage. (refs: https://github.com/confident-ai/deepeval, https://deepeval.com/enterprise, https://github.com/Giskard-AI/giskard-oss, https://www.giskard.ai/, https://docs.evidentlyai.com/faq/oss_vs_cloud)
- **plan**: Use two visible scoring layers: Evaluation Fit Score for RAG evaluation capability and Operational Fit Score for deployment, governance, and production readiness. Suggested weights: RAG metrics 20%, CI/regression 15%, dataset/testset workflow 15%, observability/tracing 15%, integration fit 10%, deployment/data control 10%, access control/retention/audit 10%, public evidence maturity 5%. (refs: https://docs.ragas.io/en/stable/, https://deepeval.com/docs/getting-started, https://arize.com/docs/phoenix/cookbook/evaluation/evaluate-rag, https://docs.langchain.com/langsmith/evaluate-rag-tutorial, https://www.promptfoo.dev/docs/guides/evaluate-rag/, https://docs.langchain.com/langsmith/enterprise, https://langfuse.com/docs/administration/rbac)
- **risk**: Evidently managed SaaS availability should be marked as conflicting public evidence / vendor confirmation required. Official docs simultaneously say Evidently Cloud is no longer available as SaaS, describe Cloud as a recommended hosted managed option, and say new users are enrolled in Cloud v2. (refs: https://docs.evidentlyai.com/faq/oss_vs_cloud, https://docs.evidentlyai.com/faq/cloud_v2, https://docs.evidentlyai.com/docs/setup/self-hosting)
- **plan**: Add explicit workbook value states such as Strong, Moderate, Limited, Framework-native, Enterprise-only, Unknown/public evidence not found, and Deprecated/legacy so caveats are visible instead of hidden inside a blended score. (refs: https://developers.openai.com/api/docs/guides/evals, https://docs.evidentlyai.com/faq/oss_vs_cloud, https://docs.wandb.ai/weave/guides/platform/weave-self-managed)
